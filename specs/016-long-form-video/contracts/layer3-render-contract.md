**Layer**: Rendering (Layer 3 — this MoneyPrinterTurbo fork)
**Endpoint**: `POST /api/v1/videos` (existing — no new route)

Mode 3 reuses Layer 3's existing video controller. The only contract delta is the `mode = "long"` literal value plus a few `VideoParams` fields the long-form mode reads.

## Mode registry entry — `app/services/modes/long_form.py`

The registry entry MUST declare:

```python
LONG = ModeConfig(
    id="long",
    aspect_ratio="16:9",
    target_resolution=(1920, 1080),
    duration_choices_seconds=(120, 180, 240, 300),
    default_duration_seconds=180,
    word_budget_per_minute=150,
    subtitle_band_y_pct=0.80,
    subtitle_text_color="#FFFFFF",
    subtitle_band_color="#000000",
    subtitle_band_opacity=0.60,
    segment_count_range=(8, 25),
    music_volume_db=-18,         # vs Mode 2's -14 (long-form prefers quieter BGM)
    script_template="HOOK_BODY_SUMMARY",
)
```

`script_template` is consumed by `generate_long_form_script` to build the LLM prompt. The exact `ModeConfig` shape lives in `app/services/modes/__init__.py` (introduced by spec 015's PR-A).

## `VideoParams` extensions

The fields long-form reads from the existing Pydantic model:

| Field | Required | Notes |
|---|---|---|
| `mode` | yes | MUST be `"long"`; the rest of the schema continues to support `"short"`, `"faceless"`. |
| `video_aspect` | yes | MUST be `"16:9"` for `mode = "long"`; controller validates and rejects mismatched combinations. |
| `video_clip_duration` | yes | Layer 2 fills with the median segment duration (≈ 12–25s). |
| `video_count` | yes | Layer 2 fills with the segment count. |
| `subtitle_position` | yes | Layer 2 fills with `"bottom"`; the mode registry's `subtitle_band_y_pct` is the source of truth at render time. |
| `voice_name` | yes | Voice id from the shared library. |
| `bgm_type` | yes | One of `"random"`, `"custom"`, `"none"`. Layer 2 maps `music_id` → `"custom"` + `bgm_file`. |
| `script` | yes | Final script (already generated by Layer 2 — Layer 3 does NOT re-call the LLM for `mode = "long"`). |
| `video_materials` | yes | Pre-signed B-roll URL list. Order matches the script segments. |

`tenant_id`, `user_id`, `mode`, `product_id` continue to be required by Principle III; demo values land here in v1.

## Behaviour — Layer 3 render path for `mode = "long"`

1. Validate `aspect == "16:9"` and `mode == "long"` consistency. On mismatch, return HTTP 422 `{"error_code": "mode_aspect_mismatch", ...}`.
2. Skip LLM script-generation (Layer 2 already produced it). The `params.script` field is the final narration.
3. Synthesize voice via existing `app/services/voice.py` path (Azure or Edge per `voice_name`).
4. Stitch B-roll to script segments using the `video_materials` URL order; per-segment duration follows `video_clip_duration`. Crossfade transitions; **no Ken-Burns zoompan** (per research D-3 — too slow).
5. Burn subtitles using the registry's `subtitle_band_y_pct` and color/opacity.
6. Mix BGM at `music_volume_db = -18` if `bgm_type != "none"`.
7. Emit `final-1.mp4` to the existing `storage/tasks/<task_id>/` directory.
8. Return the existing video-controller response shape (task id + final URL); Layer 2 picks up the file at the known path.

## Failure surfaces (Layer 3 → Layer 2)

Layer 3 returns its existing error codes (`render_failed`, `voice_failed`, etc.). Layer 2 maps them to its `assembly_failed` / `voice_synthesis_failed` taxonomy before surfacing to Layer 1. No new Layer 3 error codes introduced.

# Implementation Plan: Mode 3 long-form 10-minute cap + URL-expiry resilience + WebM selfie uploads

**Branch**: `019-longform-10min-fix` | **Date**: 2026-05-09 | **Spec**: [spec.md](./spec.md)

**Note**: This is a **retroactive plan** — the work was committed before SpecKit artifacts were authored (commits `c124ebb` and `e817809`). The plan documents the as-built design so `/speckit.tasks` can derive a verification-and-hardening task list, and `/speckit-analyze` has artifacts to compare against.

## Summary

Three paired changes shipped on this branch, scoped tightly to existing approved fork surfaces:

1. **Mode 3 cap raise (5→10 min)** — extend the long-form mode's duration choices and segment-count envelope. Pure data-table change in `app/services/modes/long_form.py`. Pairs with separate L1/L2 commits that bump the wizard's poll timeout (40→75 min) and the orchestration validator (`_VALID_DURATIONS`, `_MAX_SCRIPT_WORDS`, `shot_count` formula).
2. **URL-expiry resilience** — in `app/services/material.py:_download_from_pre_signed_urls`, treat HTTP 403/410 as a dead signature (skip the retry budget) and write a same-duration black-frame MP4 placeholder via a new `_write_black_frame_clip` helper that uses ffmpeg's `lavfi color` filter. Audio + subtitle alignment downstream is unchanged.
3. **WebM selfie upload fix** — in `app/controllers/v1/uploads.py:upload_selfie`, strip MIME parameters before lookup (so `video/webm;codecs=vp8` matches the `video/webm` allow-list) and re-encode WebM/MKV to H.264 (yuv420p, no audio) on the persist step instead of stream-copying (which silently fails because VP8/VP9 cannot live in an MP4 container).

## Technical Context

**Language/Version**: Python 3.11/3.12 (per Constitution §Technology Constraints)
**Primary Dependencies**: FastAPI (controllers), MoviePy + FFmpeg (assembly), `requests` (pre-signed URL fetch). FFmpeg `lavfi color` filter for the new placeholder helper — no new Python dependency.
**Storage**: Per-task working directory under `storage/tasks/<task_id>/clips/`. Selfie speaker references under the existing Mode 4 hybrid-persistence slot model (last-3-per-tenant, per spec 018 FR-014).
**Testing**: `pytest test/` per Constitution. New checks land under `test/services/` (material) and `test/controllers/` (uploads).
**Target Platform**: GPU-capable host (RunPod or equivalent) via the GPU Dockerfile; M-series local dev via `webui.sh` + `main.py`.
**Project Type**: Web service (Layer 3 rendering engine) — sits behind Layer 2 orchestration (`visualai-orchestration`) and Layer 1 wizard (`visualai-frontend`).
**Performance Goals**: 10-minute Mode 3 render completes in ≤75 minutes wall-clock on M-series (calibration target: 50–70 min typical). Black-frame placeholder adds no measurable latency vs. a real fetched clip.
**Cost expectation** (not a hard gate at L3, observed at L2 cost dashboard): Twelve Labs spend per 10-min render is expected to land in the $0.50–0.65 range, well under the existing $1 budget envelope. No L3 enforcement; this is an expectation derived from per-segment-API cost × ~30–40 segments.
**Constraints**: Constitution §I (no business logic in this layer), §II (changes confined to the six fork-surfaces), §III (tenant context preserved through every code path that touches a render or upload), §IV (no direct external generation-API calls — Pexels exception unaffected here).
**Scale/Scope**: ≤40 segments per render at the 10-min cap. Per-tenant Mode 4 selfie store remains 3 references max.

## Phase 0 — Research (Outputs)

See [research.md](./research.md) for full notes. Headlines:

- **Black-frame placeholder format** — chose 1280×720 @ 30fps, libx264 veryfast, no audio. Rejected: matching the failed segment's exact resolution (no benefit; assembly downscales anyway) and including a silent audio track (risks duplicate-track confusion at concat time).
- **403 vs 410 handling** — short-circuit the retry budget on both. 403 is the canonical pre-signed-URL-expired response across S3-compatible providers; 410 is rare but unambiguous. Rejected: retrying once on 403 (no signature regeneration available at L3, retry is wasted budget).
- **WebM → MP4 conversion** — re-encode video to H.264 yuv420p, drop audio. Rejected: stream-copy (`-c copy` fails for VP8/VP9 in MP4 container) and keeping audio (MuseTalk synthesises narration from the script — selfie audio is unused and risks downstream demuxer confusion).
- **Segment count upper bound** — 40 chosen to land between the spec's 12–15s/shot pacing × 600s = 40–50 segment range. Rejected: 50 (overshoots Twelve Labs budget at edge cases) and leaving 25 (creates artificial visual repetition at the 10-min target).
